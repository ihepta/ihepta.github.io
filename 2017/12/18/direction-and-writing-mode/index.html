direction

简介

该属性用来设置块级元素中的内容排布方向，可选属性值有：

ltr：从左到右，默认值

rtl：从右到左

inherit：继承自父元素

兼容性

chrome safari firefox opera ie
2.0+ 1.3+ any 9.2+ 5.5+

使用方法

对于内联元素块

首先了解一下何为“内联元素块”？

内联元素块：包括替换元素，如img、input、button、video、object等或者inline-block级别的元素。

针对行内元素，默认情况下，我们平时写html时，DOM在前的元素显示在左边，DOM在后的元素显示在右边，而如果设置了direction: rtl ，DOM在前的就会显示在右边，在后的就会显示在左边。同时<div dir="rtl"></div>也有相同的功效，如下所示：

对于文本

首先文本朝哪个方向流动取决于html的lang属性、浏览器本地的设置以及具体元素的font-family，比如阿拉伯语、希伯来语就是从右往左书写。此时需要引入一个概念unicode-bidi，一般当一行中既有向左流动的文字，又有向右流动的文字的时候就需要unicode-bidi的介入。它可以决定嵌入层的文字流向，或者覆盖默认的unicode双向算法。unicode-bidi可选属性值有：

normal：浏览器默认的展现形式，一行中只有从左往右的文字或只有从右往左的文字

embed：必须用在inline元素上，可以实现在RTL文本中存在LTR文本，取决于direction属性的设置

bidi-override：当运用到inline元素上时，和embed效果相同；当运用在块级元素上时，它会覆盖浏览器默认的双向文本算法，使块级元素内部所有的行级子元素根据设置的direction属性排列

writing-mode

简介

该属性决定了内容排版方向，使用它可以很方便地排版出从上到下，从右到左的内容，常用的场景为一些亚洲语系，比如中文的古诗词的书写形式，也可以用来做一些艺术上的展现形式。可选标准的css3属性值：

horizontal-tb：默认的属性值，内容水平从左往右排列，下一行出现在当前行下面

vertical-rl：内容垂直从右往左排列，下一行出现在当前行左边

vertical-lr：内容垂直从左往右排列，下一行出现在当前行右边

针对IE8+，可选属性值：lr-tb、tb-rl、tb-lr，分别对应以上三个属性值，如要查看IE其他版本属性值可点击这里

区别与联系

direction结合unicode-bidi可以实现行内元素的流向控制;而writing-mode可以直接改变以往我们认知的块级元素的纵横规则，用法和功能都强大很多，只要稍微创新一下就能实现很多不可思议的效果。两者的属性值可能比较相近，但是没有一点关系，各司其职而已。
